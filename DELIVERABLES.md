# DELIVERABLES.md

## Purpose

This file describes the Owlbox deliverable files and the method for placing them into a project repository.

Owlbox deliverables are the continuity files and supported implementation files. The Codex Owlbox skill is a tool for creating and maintaining those files, but the skill is not required for an Owlbox to exist. An Owlbox may also be manually built from the templates, generated by script, or seeded into an assistant as project context.

An Owlbox can exist in an unsupported form as an `owlbox/` folder containing the three OWL files. A project is Owlbox-enabled only when it has the supported implementation: the `owlbox/` folder, the three OWL files, generated `OWLBOX.md`, and a documented build or maintenance method.

## Deliverables

Owlbox provides:

- `OWLBOX.md`
- `owlbox/OUTLINE.md`
- `owlbox/WISDOM.md`
- `owlbox/LEGACY.md`

The OWL files are only:

- `owlbox/OUTLINE.md`
- `owlbox/WISDOM.md`
- `owlbox/LEGACY.md`

`OWLBOX.md` is the generated combined view of the OWL files. It is an Owlbox deliverable, but it is not itself an OWL file.

## Destinations

For a project repository:

- create or update `owlbox/OUTLINE.md`;
- create or update `owlbox/WISDOM.md`;
- create or update `owlbox/LEGACY.md`;
- place generated `OWLBOX.md` in the project root.

Use the templates in `templates/` when creating a fresh Owlbox.

Installed Owlbox deliverables create the Owlbox for the containing project. They do not create a second Owlbox inside the installed Owlbox file set.

A cloned or embedded repository may keep its own Owlbox inside that repository, because that Owlbox describes the cloned or embedded repository's scope.

## Activation

Owlbox activation is supported file presence and maintenance. A project is Owlbox-enabled when the OWL files exist in the project, generated `OWLBOX.md` exists in the project root, and a documented build or maintenance method keeps them current.

After changing canonical OWL files, run `scripts/build-owlbox.ps1` from the project repository root to regenerate `OWLBOX.md`.

## Verification

Verify that:

- the project contains `owlbox/OUTLINE.md`;
- the project contains `owlbox/WISDOM.md`;
- the project contains `owlbox/LEGACY.md`;
- root `OWLBOX.md` exists;
- root `OWLBOX.md` contains the current canonical Owlbox source content.

## Growth

Owlbox deliverable files are presented directly because Owlbox may be cloned and altered, and because a project may grow beyond its first initialization needs.
